Gra w chaos na kwadracie i pewna paproć¶

W niniejszym notatniku zakodujemy gr w chaos oraz wyrysujemy pikne grafiki powstajce podczas zajmowania si ukadami chaotycznymi.

Uywam tutaj jdra SageMath posiadajcego wiele wygodnych i poytecznych funkcji, dziki czemu nie musz ich importowa z rnych biblitek.

Bardzo skrtowo gra w chaos polega na tym, e losujemy punkt pocztkowy z pewnego wielokta na paszczynie lub w przestrzeni (u nas kwadratu jednostkowego na paszczynie), nastpnie losujemy jeden z wierzchokw tego wielokta i zaznaczamy nowy punkt na rodku odcinka czcego stary punkt z wylosowanym wierzokiem. Procedur t powtarzamy wiele razy i zaznaczamy kolejno powstae punkty na wykresie patrzc jak wyglda ich zbir - czy nie przyjmuje jakiej ciekawej postaci. Okazuje si, e w przypadku takiej gry te zbiory punktw ukadaj si w bardzo ciekawe ksztaty.

Gr moemy rwnie urozmaici nakadajc dodatkowe warunki na to, jak losowane s kolejne punkty. Warunki te opisane s w dalszej czci tego notatnika. Mona rwnie przeczyta o nich rwnie na angielskiej Wikipedii https://en.wikipedia.org/wiki/Chaos_game

Przejdmy do implementacji gry w chaos.

Ponisza funkcja losuje jedn z liczb od 1 do 4, jeli argument ${\tt bez} = 0,$ natomiast jeli ${\tt bez} = x,$ dla $x \in \{1,2,3,4\},$ to funkcja losuje liczb ze zbioru $\{1,2,3,4\} \backslash \{x\},$ czyli ze zbioru $\{1,2,3,4\}$ bez x, std nazwa argumentu opcjonalnego.

def losuj_wierzcholek(bez = 0):
    if bez == 0:
        return randint(1,4)
    else:
        lista = [1,2,3,4]
        lista.remove(bez)
        return choice(lista)

Funkcja losujca punkt z kwadratu z rozkadu jednostajnego na tym kwadracie.

In [21]:
def losuj_punkt():
    return (uniform(0, 1),uniform(0, 1))

Funkcja ${\tt nowy_\_punkt}$ przyjmuje jako argument M, bdcy punktem z kwadratu jednostkowego. Losuje ona jeden z czterech wierzchokw kwadratu i zwraca nowy punkt, ktry ley na rodku odcinka lecego midzy podanym punktem M oraz wylosowanym wierzchokiem kwadratu.

Przykadowo, jeli M = (0.5, 0.5) oraz wylosujemy wierzchoek (1,1), to funkcja zwrci nowy punkt (0.75, 0.75).

Dodatkowo, funkcja "zapamituje" ktry wierzchoek zosta wylosowany w danym momencie umieszczajc numer wylosowanego wierzchoka na drugim miejscu zwracanej listy.

def nowy_punkt(M, bez = 0):
    
    wierzcholek = (0,0)
    nr = losuj_wierzcholek(bez)
    
    if nr == 1:
        wierzcholek = (1,0)
    elif nr == 2:
        wierzcholek = (1,1)
    elif nr == 3:
        wierzcholek = (0,1)
    # jezeli nr == 4 to zostaje (0,0) 
    
    return( [( (M[0]+wierzcholek[0])/2 , (M[1]+wierzcholek[1])/2 ), nr ])

Funkcja f przyjmuje nastpujce argumenty:

M - punkt pocztkowy z kwadratu jednostkowego

n - liczba iteracji, czyli liczba punktw ktre powstaj w wyniku "grania w gr w chaos"

warunek - wariant gry w ktr gramy, czyli naoenie ograniczenia na wybr wierzchoka w kolejnej iteracji (szczegy s opisane niej)

bez - jak w funkcji ${\tt losuj_\_wierzcholek}$, z ktrej korzysta funkcja f.

Warunki odpowiadaj rnym wariantom gry w chaos. W grze z warunkiem rwnym 2 jeli nowy punkt powsta przez przesunicie starego punktu w stron wierzchoka X, gdzie $X \in \{1,2,3,4\},$ to w kolejnej iteracji nie moemy wylosowa wierzchoka, ktry "jest poprzednikiem X wzgldem ruchu zgodnie z ruchem wskazwek zegara".

Aby zabrzmiao to janiej - poprzednikiem punktu (1,0) jest (0,0),

punktu (1,1) - (1,0),

punktu (0,1) - (1,1),

punktu (0,0) - (0,1).

Warunek 3 odpowiada warianowi, w ktrym jeli punkt powsta przez starego punktu w stron danego wierzchoka, to w kolejnym ruchu nie moe zosta wylosowany wierzchoek przeciwlegy do niego.

def chaos(M, n, warunek = 1, bez=0):
    
    n -= 1
    M = [M, 0]
    
    if warunek == 1:
        lista = [M[0]]
        for i in range(n):
            M = nowy_punkt(M[0], bez)
            bez = M[1]
            lista += [M[0]]
        show(points(lista, xmin=0,xmax=1,ymin=0,ymax=1,aspect_ratio=1))

    if warunek == 2:
        lista = [M[0]]
        for i in range(n):
            M = nowy_punkt(M[0], bez)
            bez = ( (M[1]+2) % 4)+1
            lista += [M[0]]
        show(points(lista, xmin=0,xmax=1,ymin=0,ymax=1,aspect_ratio=1))
        
    if warunek == 3:
        lista = [M[0]]
        for i in range(n):
            M = nowy_punkt(M[0], bez)
            bez = ( (M[1]+1) % 4)+1
            lista += [M[0]]
        show(points(lista, xmin=0,xmax=1,ymin=0,ymax=1,aspect_ratio=1))

Paproć Barnsley'a¶

Przyjrzyjmy si jeszcze jednemu fraktalowi, a mianowicie paproci Barnsley'a. Najpierw definiujemy cztery proste funkcje, ktre przeksztacaj punkt na paszczynie w inny punkt na paszczynie, a nastpnie funkcj g, ktra z punktu pocztkowego za pomoc funkcji f1,f2,f3,f4 tworzy list punktw, powstaych przez stosowanie tych funkcji najpierw do punktu pocztkowego, potem do tak powstaych punktw i tak dalej n razy.

Funkcja ta ma zoono obliczeniow rzdu 4^n, wic nie naley przesadza wiekoci liczby n. Z reszt ju dla maych n, np. n=10 otrzymujemy bardzo adny fraktal.

In [31]:
def f1(punkt):
    x= punkt[0]
    y=punkt[1]
    return (0.85*x+0.04*y,-0.04*x+0.85*y+1.6)
def f2(punkt):
    x= punkt[0]
    y=punkt[1]
    return (-0.15*x+0.28*y,0.26*x+0.24*y+0.44)
def f3(punkt):
    x= punkt[0]
    y=punkt[1]
    return (0.20*x-0.26*y,0.23*x+0.22*y+1.6)
def f4(punkt):
    x= punkt[0]
    y=punkt[1]
    return (0,0.16*y)

def g(punkt):
    return [f1(punkt),f2(punkt),f3(punkt),f4(punkt)]

def gn(punkt, n):
    lista = [punkt]
    for i in range(n):
        lista = [g(punkt)[i] for punkt in lista for i in range(4)]
    return lista
