Rysowanie zbioru Mandelbrota¶

Zamiemy si rysowaniem zbioru Mandelbrota i robieniem zblienia na jego wybrane fragmenty. Uywam tutaj jdra SageMath aby korzysta z wielu ciekawych funkcji bez importowania ich z bibliotek.

Funkcja generujca siatk na prostokcie [a,b]x[c,d]

dd - odlego midzy ssiednimi punktami na siatce

In [94]:
def mesh(dd): #funkcja tworzca siatk na prostokcie [-2,1]x[1,1]
    M = []    # dd - odlego midzy punktami na siatce
    xp = floor(3/dd)
    yp = floor(2/dd)
    for i in range(1,xp):
        for j in range(1,yp):
            M += [ (-2+i*dd , -1 +j*dd )]
    return(M)

def mesh2(dd, a, b, c, d): #funkcja tworzca siatk na prostokcie [a,b]x[c,d]
    M = []
    xp = floor((b-a)/dd)
    yp = floor((d-c)/dd)
    for i in range(1,xp):
        for j in range(1,yp):
            M += [ (a+i*dd , c+j*dd )]
    return(M)

def g(z, c):
    return ( z[0]**2-z[1]**2+c[0], 2*z[0]*z[1]+c[1]  )

n-krotne zoenie funkcji g

In [98]:
def gn(z,c,n):
    new = g(z,c)
    for i in range(n-1):
        new = g(new,c)
    return(new)

Norma, czy te modu liczby zespolonej z.

In [100]:
def norm(z):
    return ((z[0]^2+z[1]^2)^(1/2)).n()

Funkcja f przyjmuje za argumenty siatk M oraz liczb n oznaczajc liczb zoe ze sob funkcji g. Do zbioru Mandelbrota nale wszystkie te liczby zespolone ${\tt point}$, dla ktrych cig ${\tt gn((0,0), point, n )}$ nie zbiega do nieskoczonoci. Ponisza funkcja bierze wszystkie punkty z siatki*, oblicza n-krotne zoenie funkcji g i sprawdza, czy modu tak otrzymanej liczby zespolonej jest mniejszy od 2.

Dlaczego od 2? Poniewa jeli modu ktrej z liczb z cigu iteracji byby wikszy od 2, taki cig szybko dyby do nieskoczonoci. Jeli modu jest mnieszy od 2, to wszystko wskazuje na to, e taki cig nie ucieka do nieskoczonoci.

Po wykonaniu sprawdzenia tworzona jest nowa siatka, w ktrej przy punkcie umieszczana jest informacja czy punkt naley do zbioru Mandelbrota (1) czy nie (0).

* Tworzc lepszy algorytm wcale nie trzeba sprawdza wszystkich punktw z siatki, bo to co ciekawe dzieje si jedynie na brzegu zbioru Mandelbrota. Jednak na potrzeby tego, eby tylko zobaczy zbir Mandelbrota lub jego niewielkie przyblienia zostaniemy przy obecnej wersji.

In [101]:
def f(M, n):
    N = []
    for point in M:
        if norm(gn((0,0), point, n )) < 2:
            N += [[point, 1]]
        else:
            N += [[point, 0]]
    return(N)

Funkcja draw pracuje z siatk na prostokcie [-2,1]x[1,1], aby na grafice wida byo zbir Mandelbrota w caoci. W funkcji nastpuje sprawdzenie, ktre punkty z siatki nale do zbioru Mandelbrota, a ktre nie, a nastpnie rysowana jest grafika - punkty nalece do zbioru Mandelbrota rysowane s jako wiksze, a nienalece do niego jako mniejsze.

Zmienna ${\tt sizee}$ odpowiada za wielko punktw nalecych do zbioru Mandelbrota na wykresie, a ${\tt size_\_how_\_much}$ odpowiada za wielko punktw do niego nienalecych.

Im wiksze ${\tt sizee}$, tym wiksze kropki nalece do zbioru.

Im mniejsze ${\tt size_\_how_\_much}$ tym mniej zmniejszamy kropki nienalece do zbioru, czyli robi si one wiksze.

In [102]:
def draw(d, n, sizee, size_how_much):
    N = f(mesh(d), n)
    inMandel = [pointex[0] for pointex in N if pointex[1] == 1]
    outMandel = [pointex[0] for pointex in N if pointex[1] == 0]
    return (points(inMandel, size = sizee) + points(outMandel, size = sizee/size_how_much))
